# Zephyr Web Project

A responsive multi-page website built as a front-end web development project for a first-year academic subject. It's a frontend-only project - no backend or database - built using HTML, CSS, Bootstrap, and JavaScript.

## Overview

The site includes Home, About, Contact, Achievements, Registration, and Update pages, with responsive navigation, an image gallery, and form-based UI components.

The registration page was the most challenging part of the project - getting the form layout, validation, and structure to work properly took a fair bit of trial and error. I got through it with guidance from friends and some help from AI tools when I got stuck.

The Update page lets a user edit an existing member's details (like their name, post, description). Even though there's no backend, forms across the site trigger notifications and alerts on submission, so the experience feels complete and interactive rather than static.

## Features

- Multi-page layout with responsive navigation
- Image gallery and media integration
- Registration form to add new members
- Update form to edit existing member details
- Form submission alerts and notifications (frontend-only)
- Consistent page structure across sections

## Tech Stack

- HTML
- CSS
- Bootstrap
- JavaScript

## Notes

This is a frontend-only academic project - there's no backend, so member data isn't actually persisted between sessions. Alerts and notifications on form submission are handled entirely with JavaScript, to give the site a fully functional feel despite being static under the hood. It was built to practice layout, responsiveness, and interactivity rather than to be a production-ready site. I referred to course material, official docs, friends, and AI tools (mainly for troubleshooting the registration form) along the way.
